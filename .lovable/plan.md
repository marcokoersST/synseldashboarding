

# Plan: Add English user-story explanations under each data tile (Groeimodel dashboard)

Add a small English explanation block underneath every data tile on the Synsel Groeimodel dashboard (`/super-admin/groeimodel`), written in "As a user, I want to… so that…" format. Target reader: a business analyst on the development team who needs to understand intent, data sources, and calculation logic.

## Scope

Only the Groeimodel page (`src/pages/super-admin/Groeimodel.tsx`) and its tile components. Other dashboards untouched.

## Tiles that get an explanation block

1. **KPI — Totaal opstartinvestering** (total startup investment)
2. **KPI — Gem. tijd tot break-even** (avg months to break-even)
3. **KPI — Actief in opstartfase** (active consultants still in startup)
4. **KPI — Cohort ROI**
5. **CohortChart** — cumulative balance lines per consultant
6. **Consultant timeline table** — per-consultant lifecycle rows
7. **BreakEvenHistogram** — distribution of break-even months
8. **Opstartkosten per unit** bar chart

## Block design

A subtle bordered block at the bottom of each tile (inside the Card, below the chart/data):

```text
┌─────────────────────────────────────────────────┐
│ ℹ  For the development team                     │
│                                                 │
│ As a user (C-level), I want to see [X],         │
│ so that I can [decision/insight].               │
│                                                 │
│ Data source: groeimodelData.ts → <function>     │
│ Logic: <one-line formula or rule>               │
└─────────────────────────────────────────────────┘
```

Styling: `mt-4 pt-3 border-t border-border/50 text-xs text-muted-foreground`, with a small `Info` icon from lucide and a bold "For the development team" label. English copy only inside this block; the rest of the dashboard stays Dutch.

## Example copy (KPI — Total startup investment)

> **As a user**, I want to see the total cumulative loss generated by all consultants before they reached break-even, **so that** I can quantify how much capital the company invests in ramping up new sales hires.
>
> **Data source:** `getTotalStartupInvestment()` in `groeimodelData.ts`.
> **Logic:** Sum of `startupCost` across all consultant lifecycles, where `startupCost = |min(cumulativeBalance)|` until break-even is reached.

Each tile gets its own ~3–5 line block tailored to its metric, source function, and formula.
